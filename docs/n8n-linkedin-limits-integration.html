üìù Task 1: Update Unipile Account Status Callback

P0 - Critical ~30 mins

Workflow: Unipile Account Status Callback

Current Behavior:

  • Receives profile data from Unipile
  • Saves to research_cache table
  • Updates user_profiles.linkedin_profile_data

Required Changes:

1.1 Add Postgres Node: "Initialize LinkedIn Account" ‚ñº

Position: After saving profile data, before response

Operation: Execute Query

-- Create or update linkedin_accounts record
INSERT INTO linkedin_accounts (
  user_id,
  account_type,
  inmail_credits_remaining,
  inmail_credits_max,
  account_status,
  created_at,
  updated_at
)
VALUES (
  '{{$json["user_id"]}}',
  -- Detect account type from profile metadata
  CASE
    WHEN '{{$json["account_data"]["premium"]}}' = 'true' THEN 'premium'
    ELSE 'free'
  END,
  0,  -- Will be synced by InMail credit workflow
  0,
  'active',
  NOW(),
  NOW()
)
ON CONFLICT (user_id)
DO UPDATE SET
  account_type = CASE
    WHEN '{{$json["account_data"]["premium"]}}' = 'true' THEN 'premium'
    ELSE 'free'
  END,
  updated_at = NOW();
1.2 Add Postgres Node: "Initialize Message Quotas" ‚ñº

Position: After "Initialize LinkedIn Account"

Operation: Execute Query

-- Create message_quotas record (if not exists from user creation trigger)
INSERT INTO message_quotas (
  user_id,
  created_at,
  updated_at
)
VALUES (
  '{{$json["user_id"]}}',
  NOW(),
  NOW()
)
ON CONFLICT (user_id) DO NOTHING;
1.3 Update user_profiles.linkedin_premium ‚ñº 
UPDATE user_profiles
SET
  linkedin_premium = {{$json["account_data"]["premium"]}},
  linkedin_profile_updated_at = NOW()
WHERE user_id = '{{$json["user_id"]}}';

üìù Task 2: Replace Current Limit Checking Flow

P0 - Critical ~1 hour

Workflow: Limit Checking Flow

Current Implementation:

-- Old query - checking basic usage_tracking
SELECT
  messages_sent,
  connection_requests_sent
FROM usage_tracking
WHERE user_id = $1
  AND usage_date = CURRENT_DATE;

New Implementation:

2.1 Replace with Postgres Node: "Check Message Quota" ‚ñº

Operation: Execute Query

SELECT
  can_send,
  reason,
  wait_until
FROM check_message_quota(
  p_user_id := '{{$json["user_id"]}}',
  p_message_type := '{{$json["message_type"]}}',  -- 'connection_request', 'direct_message', 'inmail', 'open_profile'
  p_is_personalised := {{$json["is_personalised"]}}
);
2.2 Update IF Node: "Can Send?" ‚ñº

Condition: {{$json["can_send"]}} equals true

  • If TRUE: ‚Üí Proceed to Message Router
  • If FALSE: ‚Üí Return Error Response
2.3 Add Error Response Node (for FALSE branch) ‚ñº

Node Type: Respond to Webhook (or return to previous workflow)

Response Body:

{
  "success": false,
  "error": "quota_exceeded",
  "message": "{{$json["reason"]}}",
  "wait_until": "{{$json["wait_until"]}}",
  "retry_after_seconds": {{Math.floor((new Date($json["wait_until"]) - new Date()) / 1000)}}
}

üìù Task 3: Update Message Router

P1 - High ~45 mins

Workflow: Message Router

Current Behavior:

  • Checks Unipile connection status
  • Routes to appropriate send node (connection request, DM, InMail, open profile)
3.1 Add Postgres Node BEFORE Router: "Get Current Quotas" ‚ñº 
SELECT
  la.account_type,
  la.inmail_credits_remaining,
  mq.daily_direct_messages,
  mq.daily_connection_requests,
  mq.daily_inmails,
  mq.daily_open_profile,
  mq.pending_connections,
  mq.last_action_at,

  -- Calculate available capacity
  CASE
    WHEN la.account_type = 'free' THEN 100 - mq.daily_direct_messages
    ELSE 150 - mq.daily_direct_messages
  END AS dm_capacity_remaining,

  20 - mq.daily_connection_requests AS connection_capacity_remaining,

  la.inmail_credits_remaining AS inmail_capacity_remaining

FROM linkedin_accounts la
JOIN message_quotas mq ON mq.user_id = la.user_id
WHERE la.user_id = '{{$json["user_id"]}}';
3.2 Update Router Logic

Add these additional conditions to your router:

For Connection Request Route:
‚Ä¢ Existing: Connection status = 'not_connected'
‚Ä¢ NEW: AND {{$json["connection_capacity_remaining"]}} > 0
‚Ä¢ NEW: AND {{$json["pending_connections"]}} < 400

For InMail Route:
‚Ä¢ Existing: Connection status = 'not_connected' or 'inmail_preferred'
‚Ä¢ NEW: AND {{$json["inmail_capacity_remaining"]}} > 0
‚Ä¢ NEW: AND account_type != 'free'

For Direct Message Route:
‚Ä¢ Existing: Connection status = 'connected'
‚Ä¢ NEW: AND {{$json["dm_capacity_remaining"]}} > 0

üìù Task 5: Create InMail Credit Sync Workflow

P2 - Medium ~45 mins

New Workflow Name: Sync InMail Credits

Trigger: Schedule (every 6 hours) - Cron: 0 */6 * * *

5.1 Get All Users with LinkedIn Connected

SELECT
  up.user_id,
  up.unipile_account_id,
  la.account_type
FROM user_profiles up
JOIN linkedin_accounts la ON la.user_id = up.user_id
WHERE up.linkedin_connected = true
  AND up.unipile_account_id IS NOT NULL
  AND la.account_type != 'free'  -- Free accounts don't have InMail
ORDER BY up.user_id;

5.2 Loop Through Users

Use Split In Batches node with batch size 10

5.3 Fetch InMail Balance from Unipile

Node Type: HTTP Request

Method: GET

URL: https://api.unipile.com/api/v1/messaging/linkedin/inmails/balance

Headers:

{
  "X-API-KEY": "{{$env.UNIPILE_API_KEY}}",
  "account_id": "{{$json["unipile_account_id"]}}"
}

5.4 Update linkedin_accounts

UPDATE linkedin_accounts
SET
  inmail_credits_remaining = {{$json["response"]["credits_remaining"]}},
  inmail_credits_max = {{$json["response"]["credits_max"]}},
  inmail_renewal_date = '{{$json["response"]["renewal_date"]}}',
  updated_at = NOW()
WHERE user_id = '{{$json["user_id"]}}';

‚úÖ Testing Checklist

Pre-Integration Testing

  • Run migration successfully - verify all tables created
  • Test check_message_quota() function manually
  • Test increment_message_quota() function manually
  • Verify triggers active (on_user_created_quota)
  • Verify RLS policies enabled

Post-Integration Testing

Test Scenario 1: New User Connects LinkedIn ‚ñº
  • User connects LinkedIn via Unipile
  • Verify linkedin_accounts record created
  • Verify message_quotas record created
  • Check account_type is correct (free/premium)
Test Scenario 2: Send Connection Request ‚ñº
  • Generate message for prospect
  • Workflow calls check_message_quota()
  • Verify returns can_send = true
  • Send via Unipile
  • Verify increment_message_quota() called
  • Check message_quotas.daily_connection_requests = 1
  • Check connection_requests table has record
Test Scenario 3: Hit Daily Limit ‚ñº
  • Manually set daily_connection_requests = 20 for test user
  • Try to send another connection request
  • Verify check_message_quota() returns can_send = false
  • Verify reason message explains limit reached
  • Verify workflow returns error to user
Test Scenario 4: Rate Limiting ‚ñº
  • Send a message
  • Immediately try to send another
  • Verify blocked by 2-minute rule
  • Wait 2+ minutes
  • Verify can send again

üìä Monitoring & Observability

Key Metrics to Track

1. Quota Violations ‚ñº 
SELECT
  violation_type,
  COUNT(*) as count,
  DATE_TRUNC('hour', occurred_at) as hour
FROM quota_violations
WHERE occurred_at > NOW() - INTERVAL '24 hours'
GROUP BY violation_type, hour
ORDER BY hour DESC;
2. Daily Send Volume ‚ñº 
SELECT
  user_id,
  daily_connection_requests,
  daily_direct_messages,
  daily_inmails,
  last_action_at
FROM message_quotas
ORDER BY last_action_at DESC;
3. Pending Connections Health ‚ñº 
SELECT
  la.user_id,
  mq.pending_connections,
  la.connection_acceptance_rate,
  CASE
    WHEN mq.pending_connections > 400 THEN 'CRITICAL'
    WHEN mq.pending_connections > 200 THEN 'WARNING'
    ELSE 'HEALTHY'
  END as status
FROM linkedin_accounts la
JOIN message_quotas mq ON mq.user_id = la.user_id
WHERE mq.pending_connections > 0
ORDER BY mq.pending_connections DESC;
